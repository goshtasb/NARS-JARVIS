# NARS-JARVIS runtime dependencies (the daemon + work primitives).
#
# Local-first by default. The autonomous execution tier stays air-gapped (test-locked). The ONLY
# outbound network is the ADR-034 read-only web-search egress below (queries go to DuckDuckGo; no
# local files or telemetry are uploaded), and it runs in an isolated subprocess.
#
# llama-cpp-python — local GGUF inference for the 7B brain + embeddings (imported lazily).
# psutil          — system sentinel + the overnight resource monitor.
# pypdf           — pure-Python PDF text extraction for the ADR-032 read_file/summarize_file primitives.
# pdfplumber      — LAYOUT-aware PDF extraction (word bbox + font size/weight) for the triage StructureSensor
#                   (born-digital contract structure). MIT (built on pdfminer.six). Chosen over PyMuPDF,
#                   whose AGPL license is an enterprise/due-diligence hazard. (matplotlib/PIL are dev-only,
#                   used solely to generate committed test fixtures — NOT runtime/CI deps.)
# readability-lxml + beautifulsoup4 + lxml — ADR-034 web reader (extract a page's main article text).
#                   HTTP itself uses the standard library (urllib) — no requests/httpx dependency.
# truststore      — make urllib verify TLS against the macOS Keychain (so a corporate/proxy root that
#                   the browser already trusts works) WITHOUT disabling verification (ADR-034).
# playwright      — ADR-039 rendered-fetch escalation: a TRANSIENT headless Chromium for JS-rendered
#                   data sites, only inside the isolated egress subprocess. After install, run
#                   `python -m playwright install chromium` once. Optional: without it, the web layer
#                   degrades to the static (urllib) tier.
